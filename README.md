# medcost-prediction-app
Прототип системы прогнозирования годовых медицинских расходов пациента на основе табличных данных

## 📖 О проекте

**MedCost Prediction App** — это учебный hi-fidelity прототип цифрового ML-продукта, который позволяет ввести данные пациента, получить прогноз ожидаемых медицинских расходов и сохранить расчёт.

Проект ориентирован на сценарий работы **страхового аналитика**, которому нужно быстро и единообразно оценивать кейсы по данным пациента.

## 🚀 Запуск проекта

1. **Клонировать репозиторий:**

```
git clone https://github.com/horacemtb/medcost-prediction-app.git
cd medcost-prediction-app
```

2. **Выполнить команду:**

```
docker compose up --build
```

После запуска приложение будет доступно по адресу:

```
http://localhost:8501
```

## 📊 Данные

Используется синтетический датасет для прогнозирования медицинских расходов.

Источник: https://www.kaggle.com/datasets/miadul/medical-cost-predication-dataset/data

## ⚠️ Проблема

Страховым аналитикам нужно быстро оценивать ожидаемые расходы пациента на следующий год. Ручная оценка ожидаемых медицинских расходов:
- занимает время;
- зависит от субъективной экспертизы;
- плохо масштабируется;
- не всегда обеспечивает единый подход к оценке.

### Основная ЦА
- страховой аналитик

### Дополнительная ЦА

- руководитель направления.

## 💡 Продуктовая гипотеза

Если дать страховому аналитику простой цифровой инструмент, который по данным пациента прогнозирует ожидаемые годовые медицинские расходы, то скорость и точность первичной оценки кейсов вырастут.

## 🧠 ML-гипотеза

Демографические признаки, образ жизни, хронические заболевания, история обращений к врачам и прошлые расходы позволяют предсказывать годовые медицинские затраты с приемлемой точностью.

## 🎯 Цель проекта

Разработать прототип ML-продукта, который:
- принимает параметры пациента;
- строит прогноз годовых медицинских расходов;
- показывает результат в понятном интерфейсе;
- сохраняет историю расчётов;
- демонстрирует связку: данные → модель → приложение.

## 🧪 Формат результата

Проект рассматривается как **hi-fidelity prototype**, а не полноценный MVP.

Почему:
- нет реальной рыночной валидации;
- нет платящих пользователей;
- нет продуктовых метрик;
- данные используются в учебных целях.
